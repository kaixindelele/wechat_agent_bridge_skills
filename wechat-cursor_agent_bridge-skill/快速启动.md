# 微信 ↔ Cursor Agent 桥接：快速启动

本教程帮你**从 0 到能在微信里和本地 Cursor Agent 对话**。

---

## 一、你需要准备什么

1. **电脑能上网**，且已安装 **Node.js 18 或以上**（终端输入 `node -v` 能看到版本号即可）。
2. **Cursor CLI（agent 命令）**：这是 Cursor 提供的命令行 Agent 工具，使用你的 **Cursor 订阅**额度（不需要 Anthropic API Key）。
   - 安装方式：`curl https://cursor.com/install -fsSL | bash`
   - 安装后终端输入 `agent --version` 能看到版本号即可。
   - 如果在无桌面的远程服务器上使用，需设置 `CURSOR_API_KEY` 环境变量（在 Cursor 账户设置中获取）。
3. **微信侧**：通过 **ClawBot / ilink** 扫码绑定机器人（本项目的 `npm run setup` 就是扫码流程）。

---

## 二、5 步启动

### 第 1 步：进入目录并安装依赖

```bash
cd wechat-cursor_agent_bridge-skill/templates
npm install
```

### 第 2 步：复制并编辑配置

```bash
cp bridge.config.example.json bridge.config.json
```

用编辑器打开 **`bridge.config.json`**，**至少改一项**：

| 字段 | 说明 |
|------|------|
| **`cwd`** | 改成你希望 Agent **读写代码的文件夹**（建议与 Cursor 打开的项目根目录一致） |

其他可先不动。需要换模型时再改 `model`（留空则用 Cursor 默认模型）。

### 第 3 步：确认 Cursor CLI 可用

```bash
agent --version
```

能看到版本号即可。如果提示找不到命令，请先安装 Cursor CLI。

### 第 4 步：扫码绑定微信（只需做一次）

```bash
npm run setup
```

- 终端里会出现二维码，用 **微信**扫码并按提示确认。
- 成功后会生成 **`credentials.json`**（不要泄露、不要提交到 Git）。

### 第 5 步：启动桥接

```bash
npm start
```

- 窗口要**一直开着**；关掉就等于停止服务。
- 之后在**微信机器人对话**里发消息，就会由本机 Cursor Agent 处理。

---

## 三、你能用微信做什么

| 功能 | 说明 |
|------|------|
| **文字对话** | 直接发文字，Agent 在你设的 `cwd` 里读代码、查文件、执行工具等 |
| **发图片** | 图片会保存到本地，Agent 可通过文件路径查看 |
| **发文件/语音/视频** | 落到本机缓存目录，消息里带上路径 |
| **会话记忆** | 默认在一定时间内接着上次对话（超时或 `/clear` 会新开） |

### 微信里的命令

| 消息 | 作用 |
|------|------|
| **`/help`** | 简短帮助 |
| **`/clear`** | 清空当前会话，下一句为新对话 |

## 四、配置说明

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `cwd` | 用户主目录 | Agent 的工作目录 |
| `agentPath` | `"agent"` | Cursor CLI 路径（如果不在 PATH 里需写完整路径） |
| `model` | `""` | 模型名称，留空用 Cursor 默认 |
| `force` | `true` | `--force` 模式：Agent 自动执行所有工具，无需逐个确认 |
| `agentTimeoutMs` | `300000` | 单次请求最长运行时间（毫秒），超时终止 |
| `maxMessageLength` | `4000` | 微信单条消息最大长度，超长自动分段 |
| `enableSession` | `true` | 是否启用会话续接 |
| `sessionTimeoutMs` | `1800000` | 会话续接超时（毫秒） |
| `showToolCalls` | `true` | 是否在微信里显示 Agent 调用了什么工具 |
| `verbose` | `false` | 详细日志 |
| `allowedUserIds` | `[]` | 允许使用的微信用户 ID 白名单（空=不限制） |

## 五、常见问题

### 启动与环境

1. **`agent` 命令找不到**
   - 先安装：`curl https://cursor.com/install -fsSL | bash`
   - 或检查 PATH 是否包含 `~/.local/bin`

2. **远程服务器无 Cursor 桌面**
   - 需设置 `CURSOR_API_KEY` 环境变量
   - 或在有 Cursor 的机器上先登录，CLI 会缓存认证信息

3. **`cwd` 填错或没权限**
   - Agent 读不到你的项目或无法写文件
   - `bridge.config.json` 里 `cwd` 改成真实存在且有读写权限的目录

4. **这不是「Cursor IDE 内的插件」**
   - 桥接是独立 Node 程序，通过 Cursor CLI 调用 Agent 能力
   - 与 Cursor 图形界面的 Agent 是独立的进程

### 微信与网络

5. **扫码失败**
   - 检查网络能否访问微信 ilink 域名；公司网络可能需要代理

6. **上一条还没跑完**
   - 会提示稍等；不要连续发多条

### 安全

7. **`force: true` 意味着什么？**
   - Agent 会自动执行所有工具（读写文件、运行命令）而不需要逐个确认
   - 如果要更谨慎，改为 `false`（但 headless 模式下部分工具可能无法执行）

8. **不要用不可信的仓库路径**
   - Agent 能读写 `cwd` 下的文件、执行命令

## 六、文件说明

| 文件 | 作用 |
|------|------|
| `templates/bridge.config.json` | 工作目录、模型等配置 |
| `templates/credentials.json` | 微信登录凭据（扫码生成） |
| `templates/bridge-state.json` | 运行后自动生成：消息游标、会话续接 |

换电脑或清空状态：重新 `npm run setup` 扫码即可。
